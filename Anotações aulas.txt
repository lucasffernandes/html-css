  - O código javaScript pode ser escrito dentro da tag <script>.

  - Após criar a função, basta colocar o nome dessa função (Chamar a função) dentro da definição CSS {onclick}. 
     Ex: onclick = funcaoJS()

* Em javaScript '=' significa "receber"
  - E '==' significa "igual"

* Como o icone é inserido como uma fonte do google fonts, como uma fonte ele é considerado, então como tal o alinhamento é realizado como elemento textual (com 'text-align')

* Com a configuração CSS 'cursor:pointer' o cursor é modificado para ser uma mão quando posicionado em cima de um objeto.

Capítulo 26 Aula 06 – Primeiros passos com Flexbox CSS
--------------------------------------------

* Flexbox é um conceito da nova versão da CSS que visa rescolver problemas relacionados a responsividade
      - flexbox (ou flexible Box) é om módulo da nova versão css 3.O

* Nessa nova versão css tambem tem o recurso de Grid Layout

CSS 2.1

-> Block Layout
-> Inline Layout
-> Table Layout
-> position Layout

CSS 3.0

-> CSS flexible bOX (flexbox) Module Level 01

-> CSS Grid Layout Module Level 1/2

--------

* Não quer dizer que os recursos da CSS level 2.1 não sejam úteis mais, eles ainda podem ser utilizados.

* o flexbox organiza os itens dentro da caixa(Conteiner) filha. ou seja, elá é um recurso adicionado geralmente a caixa pai.

* Para esse recurso funcionar é necessário colocar 'flex: auto' na caixa filha para que funcione o recurso flexbox.

   - organize automaticamente os recursos dentro da caixa de acordo com o tamanho do conteiner.

* somente recebe essas configuracoes os conteiners(Flex-conteiner), os itens(flex-itens) recebem as configurações individuais na css.

Capítulo 26 Aula 06 – Direções e Eixos Flexbox CSS
--------------------------------------------

Eixos (axis)

* Alem da configuração flexbox no container pai, existe outras definições desse recurso(flexbox) disponíveis para utilizar no posicionamento de conteiners. 
   
   - Dentre elas a flex-direction, é uma definição que vem por padrão o valor ":row" - que quer dizer linha -, ou seja, já vem pré configurado a organização das caixas em linha. Nessa configuração os ítens vão ser configurados da esquerda para direita.

      - existe outro valor para a propriedade flex-direction, que seria ":row-reverse", nesse caso, o sentido de organização dos elementos será dá direita para a esquerda.

   - existe tambem o flex-direction: colum; -> que faz a organização de ítens em coluna, de cima para baixo

      - exite nesse caso também o valor colum-reverse; que organiza o sentido dos itens de baixo para cima.

* Para configurar as posições dos elementos, o que precisa ser definido, é a posição dos eixos do conteiner pai, utilizando esse recurso de flexbox, toda caixa pai passa a ter eixos de localização no espaçamento dentro da caixa pai. Podemos definir os eixos como:

  eixo imaginário X = main axis (main-start, main-end)

  Eixo imaginário Y = cross axis (cross-start, cross-end)

* O sentido dos eixos mudam de acordo  com a configuração flex-direction:
   ex:

   flex-direction: row 
     cross-axis (y) cima ->  baixo 
     main-axis (x) esquerda -> direita

   flex-direction: row-reverse
     cross-axis(y) cima -> baixo
     main-axis (x) direita -> esquerda

   flex-direction: colunm
     cross-axis (y) esquerda -> direita
     main-axis (x)  cima -> baixo

   flex-direction: column-reverse
     cross-axis (y) esquerda -> direita
     main-axis (x) baixo -> cima

RESUMINDO

* flex-direction: com o valor 'row' organiza os elementos horizontalmente da esquerda -> direita

- com o valor row-reverse, organiza esses elementos horizontalmente de maneira reversa da direita para a esquerda

* flex-direction: com o valor colunm, organiza os elementos verticalmente de baixo para cima.

- com o valor colunm-reverse, os elementos ficam organizados verticalmente, de maneira reversa, de cima para baixo.

Capítulo 27 Aula 08 – Empacotamento e fluxo no Flexbox CSS
--------------------------------------------


* flex-wrap: nowrap -> Faz com que os conteudos sejam blockeados para serem quebrados para a linha de baixo caso o conteiner seja diminuído. nesse caso, configurado assim, os elementos são encolhidos para caber dentro do conteiner pai. 
   -Obs: o encolhimento depende do tamanho do item que será configurado.
   - nowrap = não quebrar

   - é um valor que já vem padrão, caso queira outro comportamento o valor deve ser alterado.

   - caso o valor seja 'wrap' os elementos são quebrados para a baixo para caber no conteiner.
     - essa quebra é feita no sentido de acordo com a configuração do sentido do eixo transversal cross-axis(ou eixo y)

     - wrap-reverse faz a quebra no sentido contrário ao wrap, faz a quebra dos elementos caso o conteiner diminua para cima.

* Flex-flow -> é uma maniera simplificada de lidar com as propriedades flex-wrap e flex-direction.

  -é na verdade uma shorthand pra poder declarar essas propriedades.

  - Ex: se o flex-direction e o flex-wrap estiverem com o valor padrão (row/nowrap), p flex-flow ficará:
    
     flex-flow: row nowrap

PADRÃO W3C:

 -> UTILIZA-SE NO LUGAR DE flex-direction e flex-wrap o FLEX-FLOW

 * flex:auto -> é usado somente no conteiner filho
   
   - não deixa espaços brancos no conteiner pai, fazendo os elementos internos ocuparem todos os espaços do conteiner pai.


  *Flex-flow: direção fluxo-de-quebra

Capítulo 27 Aula 09 – Alinhamento nos eixos Flexbox
--------------------------------------------

* justify-content -> faz o alinhamento de elementos de acordo com o eixo principal (main-axis)

* main-axis tem o começo (main-start) no lado esquerdo do conteiner principal e termina (main-end) no final do conteiner.

* Um valor de configuração da propriedade Justify-content é o flex-start, que configura o alinhamento de elementos no começo do eixo principal, deixando a sobra de espaço pro final do eixo.

  - RELEMBRANDO -> o sentido do eixo principal, vai depender do sentido configurado como primeiro parãmetro no Flex-flow.

* Em contrapartida, o valor flex-end do parâmetro justify-content, faz com que os elementos sejam alinhados no final do eixo principal, e a sobra de espaço é armazenada no lado esquerdo do conteiner.

  - O valor center, coloca todos os elementos colados e centralizados no centro do conteiner e as sobras vai ser divididas entre as extremidades inicial(main-start) e final (main-end) do eixo principal.

* O justify-content: space-between -> faz a gestão de espaço colocando o primeiro elemento no main-start e o ultimo elemento no main-end, colados nos dois extremos.

* Justify-content: space-evenly -> faz a gestão do espaçamento distribuindo entre os elementos o espaço que restou do conteiner pai de maneira igualitária.

* Justify-content: space-around faz a gestão do espaçamento não como o espace-evenly, que tem como objetivo alinhar os itens distribuindo de maneira igual entre os elementos. o space-around faz o alinhamento levando em consideração o espaçamento entre os elementos (onde começa um elemento e onde termina). Levando em consideração isso ele faz o alinhamento de elementos.
 
* como em colunas e em configurações reversas do flex-flow os sentido do eixo principal muda, é necessário tomar cuidado quanto o sentido e a gestão de espaço de cada parâmetro.

ALINHAMENTO PARA EIXO TRANSVERSAL (cross-axis)

* Para fazer o alinhamento no pelo eixo transversal o idel é aumentar a altura do conteiner para entender como está sendo feito a gestao do espaço (no caso para aprender as configurações)

* a propriedade que faz a gestão de espaçamento transversal é
align-itens:

  - ela recebe por padrão o valor 'stretch' que faz o "esticamento dos elementos" no sentido do cross-axis.

  - Flex-start -> é o valor dessa propriedade que faz o alinhamento dos elementos no começo do eixo transversal (Cross-start) e o espaçamento restante fica concentrado no fim do eixo transversal (cross-end)

  - o espaçamento restante do lado direito dos elementos pertencem ao eixo principal, e o espaço que sobra no fim do eixo transversal pertencem ao eixo transversal.

  - align-items: flex-end -> faz o alinhamento dos elementos no final do eixo TRANSVERSAL, e o espaçamento restante fica concentrado no inicio do eixo.

  - o valor 'center' alinha os elementos no centro do eixo, deixando as sobras de espaço igualmente divididas entre as extremidades.

  - align-items não tem as configurações de espaçamento, como: space-between, space-around e space-evenly existe uma outra propriedade que faz isso para o eixo transversal.

Capítulo 27 Aula 10 – Centralização absoluta com Flexbox
--------------------------------------------

* Usando Flexbox fica bem mais facil para centralizar elementos.

RELEMBRANDO RESUMIDAMENTE

* para alinhar os elementos pelo eixo principal (main-axis) é utilizado Justify-content.

* Para alinhar os elementos pelo eixo transversal (cross-axis) é utilizado align itens.

* essas configurações de flex-box são sempre aplicadas ao conteiner pai

* Para utilizar essas duas propriedades para centralizar os elementos basta fazer dessa forma:

   Justify-content: center
   align-itens: center

* Para centralizar elementos a primeira configuração a ser feira no conteiner pai é o 'display:flex'.

* Antes de centralizar, é bom (não é via de regra) deixar definido o flex-flow para que seja descrito o fluxo do eixo principal e o transversal.

* semanticamente (não é uma regra) é melhor nao utilizar o body html como conteiner pai na centralização de elementos.

Capítulo 27 Aula 11 – Alinhamento de elementos empacotados no Flexbox
--------------------------------------------

* Justify-content -> alinha os elementos de acordo com o eixo principal

* align-itens -> alinha os elementos de acordo com o eixo transversal

* align-content -> Alinha os elementos no eixo transversal "quando estão empacotados."

 - O valor 'stretch' vai esticar o elemento até que preencha toda a linha transversal

   - Esse é o valor padrão

   - se caso os elementos forem quebrados e dois ou tres elementos esse valor divide o espaço de acordo com a quebra da linha. Ex: se for duas linhas essa configuração divide o espaço do eixo transversal em duas linhas e estica os elementos até ocuparem toda a linha delimitada.

 - O valor 'flex-start' gruda todos os elementos no cross start do eixo transversal

   - esse valor concetra todo o espaço vazio na parte de baixo do eixo transversal


 - O valor 'flex-end' aproxima todos os elementos no cross end (no final) do eixo transversal

   - Esse valor concentra todo o espaço vazio no cross start do conteiner.

  - O valor 'center' centraliza os elementos no centro do eixo transversal e os espaços vazios manda para as extremidades do conteiner

 - O valor 'space-between' coloca os elementos próximo as extremidades e divide os espaços livres centrais com os elementos do meio.  

- o valor 'space-evenly' alinha os elementos de acordo com o espaço do conteiner

- 'space-around' -> divide oseixo transversal em partes iguais, e estica eles para que ocupem totalmente o espaço delimitado

- 'align-content' só funciona quando a linha está abilitada quebra de elementos (wrap).

Capítulo 27 Aula 12- – Anatomia dos itens Flexbox
--------------------------------------------

* Somente os conteiners recebem configurações Flex,
os filhos não recebem esse tipo de configuração, eles recebem outro tipo.

  -Obs: podem até receber, mas não é bom para a anatomia do código.

* Por padrão, os conteiners são flexiveis, os ítens não são.

* 'flex:auto' é uma propíedade aplicada a ITENS.

* 'order' é outra propriedade aplicada a ITENS, por padrão o seu valor é 0.

  - Para modificar a ordem dos itens dentro de um conteiner basta atribuir um número desejado dentro de cada item com essa propriedade que eles serão ajustados com base nesses valores atribuídos.

  - Com isso a CSS vai ordenar os itens em ordem CRESCENTE.

* 'align-self' atribui a cada elemento um alinhamento individual.

 -Essa propriedade funciona no pelo eixo transversal

     -> Valores de 'align-self':
  
 - auto -> herda a característica de alinhamento vertical do seu pai.

 - flex-start -> coloca o item no cross-start do eixo transversal (no primeiro lugar)

 - flex-end -> coloca o item como último elemento do eixo transversal.

 - center -> coloca o ítem no centro do eixo transversal.

 - stretch -> estica o elemento para caber em toda a largura do eixo transversal.

Capítulo 27 Aula 13- – Propriedade flex-basis em Flexbox
--------------------------------------------

* 'Flex-basis' serve para configurar o tamanho dos itens dentro de um container

* Por padrão, o valor dessa propriedade é 'auto', e torna o tamanho do item de acordo com o seu conteúdo interno.

* se o item estiver configurado para ser em linha, o que será definido é a largura dos elementos, se caso estiver em coluna, será configurado a altura dos elementos dentro do conteiner.

outros valores de flex-basis:

 - pixels (ex flex-basis: 200px) -> define o valor em pixel para os elementos de 200 px.

    - Nesse caso, pode ser ser definido tanto o tamanho de todos os elementos, quando de elementos individuais.

    - com essa configuração, se o espaço do conteiner não for suficiente para caber os elementos em 200px eles são diminuídos em tamanhos iguais até que caibam no conteiner.
    
    - se for diminuído e o conteúdo não couber dentro, o elemento passará para fora do conteiner se for configurado o flex-flow em nowrap(sem quebra).

    - Se o flex-flow permitir quebra, com essa configuração será feito as quebras necessárias para caber os elementos dentro do eixo necessário, evitando o transbordamento.

RELEMBRANDO

 * align-content faz o alinhamento do conteúdo empacotado, ou seja, faz o alinhamento visando o conteudo empacotado (dentro) dos itens do conteiner.

Capítulo 27 Aula 14 – Controle do tamanho de itens flexíveis
--------------------------------------------

Ainda propriedade aplicada a itens e não conteiners

* 'flex:glow' configura o quanto pode crescer e 'flex:shrink' é o quanto pode encolher

* para funcionar esse recurso é preciso definir o flex-flow: como 'nowrap', pq caso contrario em vez de crescer sera quebrado o conteiner.

* Por padrão, o flex-shrink é igual a 1 e o flex-glow é igual a 0.

    Pode encolher? Pode aumentar?
  
   - 1 significa que sim / 0 significa não

   - seguindo os valores padrão: 
     
     flex-shrink:1 = pode encolher
     flex-glow: 0  = não pode crescer

   - se for travado a diminuição dos elementos com essas propriedades e o conteiner for diminuido para ser menor que o tamanho dos elementos configurados, ocorrerá o transbordamento.

   FLEX-SHRINK / FLEX-GLOW PARA VALORES DIFERENTES DE 0 E 1:

* Valor '2' -> se o flex-glow receber esse valor, o item configurado cresce o dobro do tamanho do item que estiver com o número 1.

* na ocasião de:
 
  elemento = 0 / elemento = 2 / elemento = 2

  - nesse caso, como não tem referência, os dois elementos que receberam o valor 2 vao crescer na mesma proporção. (como não tem o numero 1 como referencia de valor à ser dobrado de tamanho)

* A mesma lógica se aplica a Flex-shrink para diminuir elementos.

* geralmente as propriedades flex-basis, flex-glow e flex-shrink trabalham juntas.
